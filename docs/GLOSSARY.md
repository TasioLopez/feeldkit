# FeeldKit Glossary

- Canonical concept: neutral domain concept independent of source/target vendor.
- Canonical value: stable standardized value under a field type.
- Field pack: versioned package of field types, values, aliases, and rules.
- Field reference: explicit reuse link from one pack field to a canonical field.
- Crosswalk: mapping relationship across field types or code systems.
- Flow pack: pre-reviewed deterministic mapping profile for a common route.
- Inference: probabilistic mapping decision based on signals and learned history.
- Confidence policy: threshold logic controlling auto-apply versus review.
- Review by exception: human review only for ambiguous/low-confidence outputs.
- Provenance: origin metadata for values, aliases, crosswalks, and decisions.
- Trace: explainability payload showing why an output was selected.
